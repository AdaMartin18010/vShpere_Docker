## 📝 使用说明

### 文档组织原则

✅ **虚拟化与容器化分离** - 独立目录，便于单独维护  
✅ **硬件与软件分离** - 硬件规范独立，软件安装独立  
✅ **技术规范与实施分离** - 规范在前，实施在后  
✅ **功能模块化** - 每个功能独立文档  

### 文档命名规范

- 编号前缀：便于排序 (`01_`, `02_`, ...)
- 清晰描述：文件名直接说明内容
- 中文命名：便于中文用户理解

### 文档维护

- 单一职责：每个文档只关注一个主题
- 相互引用：文档间通过链接关联
- 版本控制：重要变更记录版本号
